# MrSample — v0.2 followups

Captured during v0.1 shipping (2026-05-16).

## Loop point granularity

`loop_start` / `loop_end` use step = 0.01 (1% of sample length). For long
samples this is too coarse to dial in usable loops by ear — a 4-second
sample steps in 40 ms chunks per knob click.

Options to evaluate:

- Drop step to `0.001` (0.1%) — still normalized 0..1, simple fix.
- Expose loop points in *frames* not normalized — needs UI to read sample
  length and clamp, but matches Live's Sampler.
- Lean on `shift_increment_multiplier` more aggressively (already 0.25);
  could go finer when shift held.

Probably do step=0.001 + keep shift multiplier; revisit frames-based later.

## Other deferrals

- Async sample loading (worker thread). Currently load is synchronous on
  the audio thread's set_param caller, which is fine for short samples
  but will stutter on large FLAC/MP3 decodes. MrDrums has the same trade.
- Pitch LFO (currently LFO routes to filter only).
- Filter envelope as a separate ADSR (currently filter env = amp env × amount).
- Multi-zone / round-robin (out of scope for "simple sampler"; that's SFZ territory).
